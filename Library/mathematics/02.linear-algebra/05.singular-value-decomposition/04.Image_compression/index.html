To truly understand how the SVD compresses data, we must start by examining the illusion of size. Matrices can be massive, but their size is often highly deceptive.

The Illusion of Size in Symmetric Matrices

Imagine a massive dataset, perhaps representing the covariance of $n = 1,000,000$ variables. The resulting symmetric matrix $S$ would be an $n \times n$ grid containing $1,000,000,000,000$ entries.

However, suppose we know that this symmetric matrix $S$ was actually generated by multiplying a much smaller feature matrix $A$ by its transpose: $S = A^TA$. If $A$ is $m \times n$, where $m = 100$ features, a profound realization occurs regarding storage:

  • Storing $S$ directly: $n^2 = 1,000,000,000,000$ values.
  • Storing $A$ instead: $mn = 100,000,000$ values.

Because $m \ll n$, we see that $mn \ll n^2$. Storing $A$ "compresses" the information of $S$. The massive matrix $S$ is mostly empty mathematical space and redundant correlations. All of its actual underlying information is entirely bottlenecked by the inner dimension $m$.

The Rank-1 DNA

We can push this logic even further. We don't just have to store $A$; we can dissect the symmetric matrix $S$ into its fundamental atomic units using its eigenvectors. Because $S$ is symmetric, the Spectral Theorem guarantees we can break it down into a sum of purely rank-1 matrices:

$$S = \sum_{i=1}^{r} \lambda_i v_i v_i^\top$$

Think about what $v_i v_i^\top$ physically is. It is a column vector $v_i$ multiplied by a row vector $v_i^\top$. If $v_i$ has length $n$, the resulting matrix is $n \times n$. But to store that entire $n \times n$ grid on a computer, you only need to save the single vector $v_i$ (which is $n$ numbers) and the scaling factor $\lambda_i$. You are generating a massive grid from a tiny handful of vectors.

The SVD Generalization

But what if our data isn't a symmetric matrix $S$? What if it is just a standard, rectangular image matrix $A \in \mathbb{R}^{m \times n}$?

We cannot use $v_i v_i^\top$ anymore, because $A$ has different dimensions for its rows ($n$) and columns ($m$). We need a vector to handle the vertical space, and a completely different vector to handle the horizontal space. This is precisely where SVD steps in. It generalizes the symmetric idea by giving us two sets of vectors: $u$ for the columns and $v$ for the rows.

$$A = \sum_{i=1}^r \sigma_i u_i v_i^\top$$

Every piece of this sum, $\sigma_i u_i v_i^\top$, is a rank-1 matrix. It is formed by a single vertical column ($u_i$) dictating the vertical pattern, and a single horizontal row ($v_i^\top$) dictating the horizontal pattern, scaled by an importance factor ($\sigma_i$).

The Single-Color Background: Why Outer Products Save Space

Let's ground this abstract math in a visual reality. Imagine an image of a person standing against a solid, single-color studio background.

In a standard digital format, the computer memorizes every single pixel blindly. If a horizontal row of pixels looks like [Background, Subject, Subject, Background], and the twenty rows below it look exactly the same, the computer stores that sequence twenty separate times. This is a massive waste of mathematical space.

A rank-1 matrix ($u_1 v_1^\top$) exploits this redundancy by separating the what from the where.

  • The Pattern ($v_1^\top$): The row vector acts as a stencil. It holds the singular horizontal pattern of the image: [Background, Subject, Subject, Background].
  • The Intensity ($u_1$): The column vector acts as the instructions. It tells us how strongly to apply that stencil on every vertical step: [1, 1, 1, 1, 1].

When you compute the outer product, you are mathematically "stamping" the row vector's pattern down the grid. As you move down the rows, the column vector dictates the brightness of the stamp. Instead of storing an $m \times n$ grid of redundant pixels, we only store $m + n + 1$ numbers (the column, the row, and the scaling weight $\sigma$). The entire grid is dynamically regenerated on the fly.

The Truncated SVD

Important clarification: Real images are almost never low-rank matrices. In fact, they typically have full rank.

What makes compression possible is not low rank, but low effective rank. This means that while there may be many singular values, most of them are extremely small.

The singular values $\sigma_1 \geq \sigma_2 \geq \dots$ decay rapidly. The first few capture the dominant structure of the image (shapes, lighting, large regions), while the remaining ones contribute only tiny corrections—fine textures and noise.

This is why we can safely discard the tail:

  • We are not assuming the image is low-rank.
  • We are approximating it as low-rank by ignoring small singular values.

In practice, we do not transmit the full matrix $A$. We transmit a low-rank approximation constructed from only the largest singular values.

In a real photograph, one rank-1 matrix isn't enough to capture the fine details (like the texture of hair or a subtle gradient of a shadow). That is why the full SVD is a sum of these matrices. The first piece ($u_1 v_1^\top$) blocks out the massive background and the basic silhouette. The second piece adds the primary shadows. The 50th piece adds the tiny textures. As we reach the tail end of the expansion, the components are essentially mathematical noise.

This leads to our strategy: throw the small terms away. We form the truncated SVD, denoted as $A_k$, by stopping the sum at $k$ components:

$$A_k = \sum_{i=1}^k \sigma_i u_i v_i^\top$$

By the Eckart-Young Theorem, $A_k$ is the best possible rank-$k$ approximation of the original image $A$. Let us look at the arithmetic of the data structures to see the savings.

Suppose our original image $A$ is $1000 \times 1000$.
Original Storage: $1000 \times 1000 = 1,000,000$ values.

Now, suppose we use the truncated SVD and keep only the first $k = 50$ components. For each of the 50 layers, we must store the singular value $\sigma_i$ (1 value), the column $u_i$ (1000 values), and the row $v_i^\top$ (1000 values). That is 2001 values per component.

Compressed Storage: $50 \times 2001 = 100,050$ values.

By keeping just 50 components, we have reduced the storage requirement to roughly 10% of the original size. When instructed, the computer multiplies the vectors to form grids and adds the 50 grids together. To the human eye, $A_{50}$ will look almost completely indistinguishable from the 1,000,000-value original.
