What the Residual Stream Actually Does

The previous article showed that a Transformer block never overwrites its input — it accumulates. Unrolling the residual updates across $L$ layers gives

$$Y^{(L)} = Y^{(0)} + \sum_{l=1}^{L} \Delta^{(l)},$$

where each $\Delta^{(l)}$ collects the increments from masked self-attention, cross-attention, and the feed-forward network at layer $l$. The final representation is the initial embedding plus every correction the network ever wrote. Nothing is discarded; the stream is strictly additive.

This is what makes deep Transformers trainable. Gradients flow unimpeded backward through the identity skip connections, bypassing every nonlinearity. But the same additive structure carries a hidden cost that only becomes visible at depth.

The Drowning Signal Problem

Because every $\Delta^{(l)}$ is added with a fixed weight of exactly $1$, the magnitude of the residual stream grows with depth. In a Pre-Norm architecture, where normalization is applied inside each sub-layer but not to the skip connection, the accumulated stream is left unscaled. After $L$ layers, its magnitude scales roughly as $O(L)$.

This creates a structural problem. Consider what happens to $\Delta^{(1)}$, the correction written by the very first layer. By layer $L$ it is one term in a sum of $L$ terms of comparable magnitude. Its relative contribution shrinks as the sum grows. A later layer cannot selectively amplify it — the skip connection offers no mechanism for choosing which past corrections to emphasize and which to ignore. The signal from early layers is progressively diluted inside an ever-growing accumulation. The paper that introduced this analysis calls it the drowning signal effect.

Empirically, a striking consequence of this is that a substantial fraction of layers in trained models can be pruned with negligible impact on performance. Stacking more layers does not always produce a commensurate gain. The network has depth but lacks the mechanism to fully exploit it.

Key Insight: The Fixed-Weight Bottleneck

Unrolling the standard residual recurrence gives

$$Y^{(L)} = Y^{(0)} + \sum_{l=1}^{L} \Delta^{(l)}.$$

Every correction $\Delta^{(l)}$ enters this sum with an implicit weight of $1$. There is no mechanism by which a later layer can assign more importance to an early correction or less to a recent one. Every layer's contribution is treated as equally relevant regardless of what information it actually carries. This is the structural limitation that grows more costly as $L$ grows.

A Familiar Transition

The problem has a structural parallel to one already solved in this series.

Recurrent networks compress all prior token information into a single hidden state as the sequence progresses. The state at position $t$ is a fixed transformation of the state at $t-1$ — there is no mechanism for position $t$ to reach selectively back to position $t-10$ without passing through every step in between. Attention solved this by replacing the fixed sequential compression with direct, weighted retrieval: each position computes a query and attends over all previous positions, assigning learned weights based on relevance.

Standard residual connections do the same thing but along the depth dimension rather than the sequence dimension. The hidden state at layer $l$ is a fixed additive accumulation of all previous layer outputs, with no ability to selectively emphasize or de-emphasize any of them. The state at layer $L$ inherits everything from layer $1$ through $L-1$ with equal weight.

The analogy is precise:

Why the Notation Uses $\alpha_{i \to l}$

The attention weight depends on two different layers:

  • $i$: the source layer being attended to
  • $l$: the destination layer performing the retrieval

The notation

$$\alpha_{i \to l}$$

therefore means:

$$\text{“how much layer } l \text{ attends to layer } i\text{.”}$$

Writing only $\alpha_i$ would incorrectly suggest that layer $i$ has one fixed global importance everywhere in the network. But different layers need different mixtures of earlier representations.

This mirrors ordinary sequence attention. In sequence attention, position $t$ computes its own attention distribution over previous positions $j$. Depth attention applies the same idea across layers:

$$\text{layer } l \text{ attends over previous layers } i.$$
Dimension Old mechanism Problem Fix
Sequence RNN hidden state Fixed compression, no selective access to past positions Attention over sequence positions
Depth Additive residuals Fixed accumulation, no selective access to past layers Attention over layer outputs

The fix is the same in both cases: replace fixed-weight compression with learned, input-dependent attention.

Attention Residuals

Instead of summing all previous layer outputs with a fixed weight of $1$, each layer computes a softmax distribution over those outputs and takes a weighted combination. The standard update

$$Y^{(l)} = Y^{(l-1)} + \Delta^{(l-1)}$$

is replaced by

$$Y^{(l)} = \sum_{i=0}^{l-1} \alpha_{i \to l}\, Y^{(i)},$$

where the weights $\alpha_{i \to l}$ are a softmax distribution over the previous layer outputs:

$$\alpha_{i \to l} = \frac{\exp(e_{i \to l})}{\sum_{j=0}^{l-1} \exp(e_{j \to l})}, \qquad \sum_{i=0}^{l-1} \alpha_{i \to l} = 1.$$

The compatibility scores $e_{i \to l}$ are computed using a single learned pseudo-query vector $\mathbf{w}_l$ for layer $l$, which attends over keys built from the previous layer outputs:

$$e_{i \to l} = \mathbf{w}_l^\top \cdot \mathrm{RMSNorm}(Y^{(i)}).$$

RMSNorm is applied to the keys so that layers with naturally larger-magnitude outputs do not dominate the softmax simply because they are larger in scale. The pseudo-query vectors $\mathbf{w}_l$ are initialized to zero, which ensures that the initial attention weights are uniform — the model begins training behaving exactly like standard equal-weight residuals and gradually learns to deviate from this as it discovers which layer outputs are most informative.

Definition: Full Attention Residuals

Let $Y^{(0)}, Y^{(1)}, \dots, Y^{(l-1)}$ be the outputs of all previous layers, where $Y^{(0)}$ is the token embedding. Full Attention Residuals replaces the additive residual update with a learned weighted combination:

$$Y^{(l)} = \sum_{i=0}^{l-1} \alpha_{i \to l}\, Y^{(i)},$$

where

$$\alpha_{i \to l} = \mathrm{softmax}_i\!\left(\mathbf{w}_l^\top \cdot \mathrm{RMSNorm}(Y^{(i)})\right).$$

The weights $\alpha_{i \to l}$ are input-dependent: different tokens in the same layer can assign different weights to the same set of layer outputs. The pseudo-query $\mathbf{w}_l \in \mathbb{R}^{d_{\mathrm{model}}}$ is a learned parameter of layer $l$, initialized to zero. The original $\Delta^{(l)}$ increments are still computed as before — attention, FFN — and added into the selected weighted base.

The weights are input-dependent in an important sense: because $Y^{(i)}$ contains token-specific information, different tokens in the same sequence can attend differently over the depth dimension. A token that needs syntactic information might weight earlier layers more; a token requiring more abstract reasoning might weight later layers more. This is the same selective-retrieval principle that attention applies across sequence positions, now applied across depth.

What the Weights Learn

Because the pseudo-query vectors are initialized to zero, training begins with uniform weights. Each layer treats all previous layers as equally important — exactly the same as standard residuals. As training proceeds, the weights diverge as the model discovers structure in the depth dimension.

Empirically, two consistent patterns emerge. First, layers tend to assign high weight to their immediate predecessor — recency matters. Second, the original token embedding $Y^{(0)}$ retains non-negligible weight throughout the network. Any layer can reach back to the raw embedding without cost, which is not possible in standard residuals where the embedding becomes progressively buried. Third, layers that attend to different inputs (the pre-attention vs. pre-FFN inputs) develop different weight profiles, suggesting that the two sub-layers of a Transformer block make different use of depth.

The Full Attention Residuals Update

To be precise about how the deltas interact with the weighted base, the full update at layer $l$ is:

$$Y^{(l)} = \underbrace{\sum_{i=0}^{l-1} \alpha_{i \to l}\, Y^{(i)}}_{\text{weighted base}} + \underbrace{\Delta_{\text{sa}}^{(l)} + \Delta_{\text{ffn}}^{(l)}}_{\text{layer increments}}.$$

The deltas $\Delta_{\text{sa}}^{(l)}$ and $\Delta_{\text{ffn}}^{(l)}$ are computed exactly as before — masked self-attention applied to the layer's normalized input, and the feed-forward network applied to the result. What changes is the base they are added to. Instead of inheriting the full uniform accumulation $Y^{(l-1)}$, the layer starts from a selectively weighted combination of everything that came before.

Unrolling across all layers now gives a different picture from the standard case:

$$Y^{(L)} = \sum_{i=0}^{L-1} \alpha_{i \to L}\, Y^{(i)} + \sum_{l=1}^{L} \left(\Delta_{\text{sa}}^{(l)} + \Delta_{\text{ffn}}^{(l)}\right).$$

The first term is the learned depth-weighted combination of all layer outputs. The second term is the standard sum of all layer increments, unchanged. Earlier corrections are no longer buried — any layer can retrieve them with non-trivial weight if they are useful.

The Memory Cost and the Block Variant

Full Attention Residuals require every previous layer output to be available when computing the current layer's weighted base. In standard training with activation checkpointing and pipeline parallelism — the normal regime for large models — many of those outputs have already been discarded and would need to be recomputed or communicated across pipeline stages. For a model with $L$ layers and hidden dimension $d$, the memory and communication cost scales as $O(Ld)$ per layer.

Block Attention Residuals address this by partitioning layers into groups. Within a block of consecutive layers, standard additive residuals are used — the same fixed-weight skip connections as always. Across blocks, the model applies depth attention: each layer attends over the summarized outputs of all previous blocks rather than every individual layer.

If there are $N$ blocks (typically around 8), the cost drops to $O(Nd)$, which is independent of total depth $L$. The block summary is the output of the last layer in the block, accumulated in the standard additive way. Layers within the current block can also attend to the partial sum of what has been computed so far in their block, preserving local information flow without requiring access to every intermediate state.

Definition: Block Attention Residuals

Partition the $L$ layers into $N$ consecutive blocks $B_1, B_2, \dots, B_N$, each of size $S = L / N$. Let $b_n$ denote the block summary — the accumulated output of all layers in block $n$:

$$b_n = \sum_{l \in B_n} \Delta^{(l)}.$$

At any layer $l$ within block $n$, the weighted base is computed over the original embedding $b_0 = Y^{(0)}$, the summaries of all completed blocks $b_1, \dots, b_{n-1}$, and the partial accumulation of the current block up to layer $l$:

$$Y^{(l)} = \sum_{k=0}^{n-1} \alpha_{k \to l}\, b_k + \alpha_{\mathrm{curr} \to l}\, b_{\mathrm{curr}} + \Delta_{\text{sa}}^{(l)} + \Delta_{\text{ffn}}^{(l)}.$$

Memory and inter-stage communication scale as $O(Nd)$ rather than $O(Ld)$, making the approach feasible under standard pipeline parallelism. Using $N \approx 8$ blocks recovers most of the benefit of full attention residuals, with the performance gap narrowing further at larger scale.

Why This Works: The Gradient Picture

Standard residual connections preserve a direct gradient path — the derivative of the loss with respect to any hidden state has an identity component that bypasses all the nonlinearities between that state and the output. This is why deep Transformers are trainable at all.

Attention residuals preserve this property while adding flexibility to the forward path. The gradient with respect to $Y^{(i)}$ now includes a term through the attention weight $\alpha_{i \to l}$ at every downstream layer $l > i$. Crucially, because the softmax weights sum to 1, the gradient path remains bounded — there is no analogue of the exploding-gradient problem that plagued vanilla RNNs. The learned weights modulate how much gradient flows through each depth connection, but the identity paths are still there.

A related empirical observation from the paper is that hidden-state magnitudes across depth are more bounded under Block AttnRes than under standard residuals. Under Pre-Norm with standard residuals, magnitudes grow monotonically with depth — the $O(L)$ growth discussed earlier. Under Block AttnRes, the selective combination at block boundaries prevents unbounded accumulation, producing a more stable forward pass throughout training.

Practical Results

The scaling-law experiments in the paper evaluate both Full and Block AttnRes against a standard Transformer baseline across five model sizes. Block AttnRes consistently achieves lower validation loss than the baseline for the same training compute. Fitting scaling-law curves to the results, Block AttnRes reaches the same loss as a baseline trained with approximately 1.25× more compute — a meaningful efficiency gain that compounds with model size.

On downstream tasks, the gains are sharpest on tasks requiring multi-step reasoning, mathematics, and code generation. A 48B parameter model trained with Block AttnRes improved by 7.5 points on GPQA-Diamond, 3.6 on mathematics benchmarks, and 3.1 on HumanEval relative to the same model without attention residuals. These are the task categories where selective access to earlier layer representations matters most — where a step of reasoning built at layer 10 needs to be retrievable cleanly at layer 40, without being diluted.

Inference latency overhead is less than 2%, making Block AttnRes a practical drop-in replacement rather than a research-only modification.

Summary: What Changes and What Does Not

The computation inside each sub-layer — masked self-attention, cross-attention, and the feed-forward network — is unchanged. The $\Delta$ increments are computed exactly as before. What changes is the base those increments are added to. Instead of a fixed uniform accumulation of all prior outputs, each layer receives a learned weighted combination of prior outputs, where the weights are input-dependent and sum to one.

This is a modification of the residual aggregation operator, not of the core attention or FFN computation. It adds one learned vector per layer (the pseudo-query $\mathbf{w}_l$), a small number of parameters relative to the total model size.

The Depth–Sequence Duality, Completed

Looking at the full arc of this chapter: RNNs solved sequence modeling through sequential state compression. Attention replaced that compression with direct weighted retrieval across sequence positions. Standard residual connections solve depth integration through fixed additive accumulation. Attention residuals replace that accumulation with direct weighted retrieval across layer outputs.

The same conceptual transition — from fixed compression to learned selective retrieval — appears twice in the architecture, once along the sequence axis and once along the depth axis. The tools are the same in both cases: a query vector, a set of keys, a softmax over compatibility scores, and a weighted combination of values.

What this suggests is that the residual connection, largely unchanged since ResNets in 2015, was playing two roles at once: a gradient highway and an information aggregation mechanism. Attention residuals split these roles. The gradient highway remains — skip connections are still present. The aggregation is handed to a learned, input-dependent mechanism that can selectively emphasize what is actually useful at each depth.
