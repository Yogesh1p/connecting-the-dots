What Recurrence Was Providing

Token embeddings carry over unchanged to the Transformer. Given a tokenized sequence $(y_1, y_2, \ldots, y_{n_{\mathrm{dec}}})$, an embedding matrix maps each token to a dense vector, producing a matrix $Y \in \mathbb{R}^{n_{\mathrm{dec}} \times d_{\mathrm{model}}}$. Throughout this article, $d_{\mathrm{model}}$ denotes the uniform representation dimension — the same $d$ used in the previous article on attention.

What recurrence naturally provided was an implicit encoding of token order. Recurrent models process tokens sequentially, so the representation at position $t$ is built from the representation at $t-1$. Through this chain, the model encodes the order of the sequence via its evolving hidden state.

Self-attention does not have this property. It processes all tokens simultaneously and is therefore permutation-invariant: it has no intrinsic notion of which token came first. Swap two tokens and the attention weights rearrange accordingly, but nothing in the mechanism resists that swap.

Removing recurrence therefore removes this implicit ordering signal. The Transformer reintroduces it explicitly by adding a positional encoding to each token embedding:

$$Y^{(0)} = \textit{TokenEmbedding} + \textit{PositionEmbedding}.$$

Each representation now encodes both the identity of the token and its position in the sequence. The exact mechanism by which positional information is constructed and integrated — sinusoidal encodings, learned embeddings, rotary encodings — will be explored in the next article.

Re-engineering the Decoder

In the recurrent architecture, the decoder updated its internal state at each step according to

$$s_t = f(s_{t-1},\, y_{t-1},\, c_t).$$

This single update bundles three distinct responsibilities: tracking the history of previously generated tokens via $s_{t-1}$ and $y_{t-1}$, retrieving relevant information from the source sequence via the context vector $c_t$, and transforming the combined signal into a representation suitable for prediction via $f$.

The Transformer preserves all three responsibilities but implements them as separate, explicit transformations applied to an entire sequence of token representations simultaneously rather than to a single evolving scalar state.

Target History

In the recurrent model, the pair $(s_{t-1}, y_{t-1})$ summarized the prefix of previously generated tokens. The Transformer replaces this with masked multi-head self-attention. Applied to decoder representations $Y$, it allows each position to attend to earlier positions while being blocked from seeing future ones:

$$H_{\mathrm{sa}} = \mathrm{MMHSA}(Y).$$

This is the causal masking mechanism developed at the end of the previous article — the same $-\infty$ mask, now lifted into the multi-head setting. Each token gathers information from all prior generated tokens in a single parallel operation.

Source Retrieval

The recurrent decoder obtained source information through the attention-weighted context vector $c_t$. The Transformer makes this role explicit with multi-head cross-attention. Decoder representations act as queries while encoder representations provide the keys and values:

$$H_{\mathrm{ca}} = \mathrm{MHCA}(H_{\mathrm{sa}},\, X_{\mathrm{enc}}).$$

Every decoder position can retrieve whatever source information is most relevant to the token being generated. We will see exactly how the Encoder builds this rich, bidirectional $X_{\mathrm{enc}}$ matrix shortly, but for now, treat it as a static memory bank of the source text. The encoder representation $X_{\mathrm{enc}}$ is fixed for the entire decoding process; only the queries change as the decoder progresses.

Local Transformation

Finally, the recurrent update function $f(\cdot)$ transformed the combined information into the next decoder state through a nonlinear mapping. The Transformer replaces this with a position-wise feed-forward network (FFN):

$$Y' = \mathrm{FFN}(H_{\mathrm{ca}}).$$
Definition: Position-Wise Feed-Forward Network

The feed-forward network (FFN) is a position-wise nonlinear transformation applied independently to each token representation. For an input token vector $\mathbf{x} \in \mathbb{R}^{1 \times d_{\mathrm{model}}}$:

$$\mathrm{FFN}(\mathbf{x}) = f(\mathbf{x}; \theta_{\mathrm{ffn}}),$$

where $f(\cdot)$ denotes a learned nonlinear mapping with parameters $\theta_{\mathrm{ffn}}$. The same FFN is applied identically at every sequence position, with no communication between positions inside the FFN itself.

Together, the three sub-layers form a single decoder block: a transformation $Y' = F(Y, X_{\mathrm{enc}})$ that maps one set of decoder representations into a richer one. Rather than evolving a single hidden state step by step, the decoder transforms the entire sequence of token representations in one forward pass.

Stacking Blocks: Repeated Refinement

Whether encoder or decoder, applying the block transformation once is not enough. Representations are stacked $L$ times, with each layer receiving the output of the previous one:

$$Y^{(0)} \rightarrow Y^{(1)} \rightarrow Y^{(2)} \rightarrow \cdots \rightarrow Y^{(L)}.$$

The motivation is hierarchical. In an early layer, masked self-attention might only resolve local relationships — the subject of a verb, the modifier of a noun. Once these local structures are encoded in $Y^{(1)}$, a second layer can reason about them: not just "what is the subject?" but "what does that subject imply about what comes next?" Each layer refines the representation constructed by the layer below, building progressively more abstract structure from the same token sequence.

The Encoder: Bidirectional Contextualization

If the decoder is an autoregressive engine that generates tokens one at a time, the encoder is a bidirectional engine that reads a complete sequence all at once. In sequence-to-sequence tasks like translation, the full source sentence is available before decoding begins. The encoder's job is to build a rich, context-aware representation of every source token so the decoder has something useful to attend to.

Because the entire source is known upfront, there is no "future" to hide. The causal mask is removed. Every encoder token can attend to every other — both forward and backward — in a single unmasked pass.

Each encoder block contains two sub-layers rather than three. There is no cross-attention because the encoder has no external sequence to attend to; it is itself the external sequence that the decoder will later consult.

  1. Multi-Head Self-Attention (unmasked): every token queries every other token in the sequence, mixing contextual information freely across all positions.
    $$H_{\mathrm{attn}} = \mathrm{MHSA}(X)$$
  2. Position-Wise Feed-Forward Network: the newly mixed representations are refined independently at each position through the same two-layer nonlinear transformation defined above.
    $$X' = \mathrm{FFN}(H_{\mathrm{attn}})$$

Just as in the decoder, a single pass is not enough to build deep representations. The encoder repeats this two-part block $L$ times:

$$X^{(0)} \rightarrow X^{(1)} \rightarrow X^{(2)} \rightarrow \cdots \rightarrow X^{(L)}.$$

Early layers resolve local structure — adjacent words form phrases, modifiers attach to their heads. Later layers resolve long-range dependencies, coreference, and overall semantic framing. By layer $L$, the output matrix $X_{\mathrm{enc}} \in \mathbb{R}^{n_{\mathrm{enc}} \times d_{\mathrm{model}}}$ contains holistic, context-saturated representations of every source token. This is what gets passed to the decoder's cross-attention mechanism.

Residual Refinement

Deep stacks of nonlinear transformations are difficult to train because gradients must propagate backward through many layers. Each nonlinearity squashes or saturates some fraction of the gradient signal, so by the time it reaches the earliest layers it may be too small to drive useful learning.

Residual networks addressed this by introducing skip connections. Instead of learning a complete replacement transformation $F(x)$, each block learns only a correction to the current representation:

$$\text{output} = x + F(x).$$

The identity path allows gradients to flow backward unimpeded: no matter how many nonlinear blocks lie between a loss and an early parameter, there is always a straight path through the skip connections that carries an unscaled gradient signal.

Transformers adopt the same principle. Rather than replacing the current representation, each sub-layer predicts a refinement that is added to it. For a decoder block, unrolling all three sub-layers:

$$Y^{(l+1)} = Y^{(l)} + \underbrace{\Delta_{\mathrm{sa}}^{(l)}}_{\text{masked self-attn}} + \underbrace{\Delta_{\mathrm{ca}}^{(l)}}_{\text{cross-attn}} + \underbrace{\Delta_{\mathrm{ffn}}^{(l)}}_{\text{FFN}}.$$

For an encoder block, there is no cross-attention term, so the update simplifies to $X^{(l+1)} = X^{(l)} + \Delta_{\mathrm{attn}}^{(l)} + \Delta_{\mathrm{ffn}}^{(l)}$. Unrolling across all $L$ decoder layers reveals the full accumulation structure:

$$Y^{(L)} = Y^{(0)} + \sum_{l=1}^{L} \bigl(\Delta_{\mathrm{sa}}^{(l)} + \Delta_{\mathrm{ca}}^{(l)} + \Delta_{\mathrm{ffn}}^{(l)}\bigr).$$

The final representation is the initial embedding plus every correction contributed across all layers and all sub-layers. Information is never overwritten — it is progressively accumulated.

Key Insight: The Residual Stream

A Transformer block never overwrites representations — it accumulates them. Attention writes a context-aware increment; the FFN writes a local nonlinear increment. Both gradients flow unimpeded backward through the skip connections, bypassing every nonlinearity. This is why deep Transformers are trainable despite their depth.

Training and Inference

Training Objective

The model is trained to predict each token in a sequence given all preceding tokens — the autoregressive factorization:

$$p_\theta(y_1, \ldots, y_{n_{\mathrm{dec}}}) = \prod_{i=1}^{n_{\mathrm{dec}}} p_\theta(y_i \mid y_{<i}).$$

Maximizing this probability is equivalent to minimizing the cross-entropy loss summed over every position:

$$\mathcal{J}(\theta) = -\sum_{i=1}^{n_{\mathrm{dec}}} \log p_\theta(y_i \mid y_{<i}).$$

Why Training Can Be Fully Parallel

In the recurrent decoder, training was sequential: computing the hidden state at step $t$ required the hidden state at $t-1$ to be ready first. This dependency chain forced the model to process positions one after another, making training on long sequences slow.

The Transformer breaks this dependency entirely. During training, the full target sequence $(y_1, \ldots, y_{n_{\mathrm{dec}}})$ is fed to the model at once. The causal mask performs the work that the sequential chain used to do: it ensures that when computing the representation at position $i$, no attention weight reaches any position $j > i$. The causal constraint is enforced by a mask applied in parallel, not by running the model step by step.

The result is that $Y^{(L)}$ contains all $n_{\mathrm{dec}}$ prefix-conditioned representations simultaneously. Every conditional distribution $p_\theta(y_i \mid y_{<i})$ is computed in a single forward pass. The loss over the entire sequence and its gradients are computed in one shot — this is what makes training on sequences of thousands of tokens tractable.

Autoregressive Inference

At inference time the situation reverses. There is no complete target sequence to feed in — the model is generating one token at a time. Given a prefix $(y_1, \ldots, y_k)$, the model computes the distribution $p_\theta(y_{k+1} \mid y_{\leq k})$, selects a token by argmax or sampling, appends it to the prefix, and runs again for position $k+2$.

Unlike training, this is inherently sequential. Each new token depends on the one just generated, so the model must run a full forward pass once per output token. The causal mask is still present — now it simply reflects that future positions are genuinely unknown rather than deliberately hidden.

Encoder–Decoder vs. Decoder-Only

The full encoder–decoder design described above targets sequence transduction: mapping one sequence to another, such as translating English to German. The encoder builds a complete contextual representation of the source; the decoder generates the target autoregressively, attending to both its own past outputs and the encoder output through cross-attention.

For tasks that are purely generative — language modelling, code generation, dialogue — there is no separate source sequence to encode. The decoder-only architecture drops the encoder and the cross-attention sub-layer entirely. Each decoder block contains only masked self-attention and an FFN:

$$\begin{aligned} Y' &= Y + \mathrm{MMHSA}(\mathrm{LN}(Y)), \\ Y'' &= Y' + \mathrm{FFN}(\mathrm{LN}(Y')). \end{aligned}$$

The model is trained to predict the next token given all previous tokens in a single unified sequence — source and target are concatenated if the task involves conditioning. GPT and its successors are decoder-only models. Their simpler architecture, combined with the efficiency of training on vast text corpora, made them the dominant paradigm for large language models.

The full encoder–decoder equations for completeness:

$$\begin{aligned} X' &= X + \mathrm{MHSA}(\mathrm{LN}(X)), \\ X'' &= X' + \mathrm{FFN}(\mathrm{LN}(X')), \\[8pt] Y' &= Y + \mathrm{MMHSA}(\mathrm{LN}(Y)), \\ Y'' &= Y' + \mathrm{MHCA}(\mathrm{LN}(Y'),\, X_{\mathrm{enc}}), \\ Y''' &= Y'' + \mathrm{FFN}(\mathrm{LN}(Y'')). \end{aligned}$$
Encoder-Only Models and BERT

A third Transformer variant — the encoder-only architecture — removes the decoder entirely. BERT (Bidirectional Encoder Representations from Transformers) is the most influential example.

Unlike autoregressive models, encoder-only models do not predict tokens strictly from left to right. Since there is no causal mask restricting attention, every token can attend to every other token in both directions during training. This allows the model to build deeply contextual, bidirectional representations of language.

Because of this design, BERT excels at understanding-oriented tasks such as text classification, named entity recognition, semantic similarity, sentiment analysis, and question answering.

Instead of next-token prediction, BERT is trained primarily using a masked language modeling objective: random tokens in a sentence are hidden, and the model learns to predict them using surrounding context from both directions.

The Full Picture

Every component in the Transformer was forced by a concrete problem. Token embeddings convert discrete symbols into vectors. Positional encodings restore the notion of order that recurrence previously provided. Masked self-attention replaces the recurrent target history without requiring sequential processing. Cross-attention retrieves information from the source sequence. The FFN replaces the nonlinear integration once performed by the recurrent update $f$. Residual connections allow these transformations to accumulate across depth without destroying earlier computations. Layer normalization stabilizes training at each stage.

The architecture therefore accumulates corrections across depth rather than overwriting state across time. Initial token embeddings enter the first layer and emerge from layer $L$ as rich contextual representations after $L$ rounds of communication through attention and local transformation through the FFN.

Somewhere in the model's $d_{\mathrm{model}}$-dimensional representation space lie directions and regions corresponding to concepts expressed in language. A token embedding provides only a context-independent starting point. The word bank begins with the same vector regardless of whether the sentence concerns finance or a river. The role of the Transformer is to move this initial vector through the representation space until it reaches a position that reflects the word's meaning in its specific context and best predicts what comes next.

The architecture does not store meanings. It computes them — dynamically, from the structure of the sequence. This process of continually transforming representations is reflected in the name Transformer.

To ground these abstract variables, the original Transformer paper used $d_{\mathrm{model}} = 512$, $L = 6$ layers for both encoder and decoder, and 8 parallel attention heads. While modern Large Language Models scale these up to $d_{\mathrm{model}} > 8000$ and $L > 80$, the fundamental machinery remains identical.

Historical Evolution of Ideas

The Transformer architecture did not emerge in isolation. It is the result of several lines of research that progressively shaped its components.

  • Recurrent Neural Networks (1990s). Early sequence models introduced processing tokens sequentially while accumulating history in a hidden state — establishing the need for contextual representations.
  • LSTM and GRU (1997–2014). Gated recurrent models addressed long-range dependencies and enabled stacked sequence models. The practice of stacking multiple layers reappears directly in the Transformer.
  • Encoder–Decoder Seq2Seq Models (2014). Neural machine translation introduced the encoder–decoder paradigm. The original Transformer preserves this high-level structure.
  • Early Attention Mechanisms (2014–2015). Bahdanau et al. replaced the fixed-size context vector with dynamically computed weighted combinations of encoder states, directly inspiring scaled dot-product and self-attention.
  • MADE and Autoregressive Masking (2015). Masked feed-forward networks demonstrated that autoregressive generation can be enforced through structural masking. Transformer decoders adopt the same principle with causal attention masks.
  • Residual Networks (2015). Skip connections enabled stable training of very deep vision networks. Transformers apply residual connections around both attention and feed-forward blocks for the same reason.
  • Layer Normalization (2016). Layer normalization stabilized deep sequence models and became a standard component of every Transformer layer.
  • Convolutional Seq2Seq Models (2017). Fully parallel sequence models using convolution demonstrated that recurrence was not strictly necessary for sequence modeling — paving the way for the Transformer's non-recurrent design.
Summary: A Synthesis of Ideas

The Transformer synthesizes stacked sequence modeling from RNNs, dynamic context retrieval from attention, autoregressive masking from feed-forward autoregressive models (including MADE, covered in the NADE article), residual learning from deep vision networks, and fully parallel computation from convolutional sequence models. No single component is novel in isolation; the contribution is how they fit together.

A Unifying View

All of the architectures discussed in this chapter arise from the same objective. Using the chain rule, the joint distribution of a sequence factors as

$$p(y_1, \dots, y_{n}) = \prod_{i=1}^{n} p(y_i \mid y_{<i}).$$

Autoregressive architectures therefore differ primarily in how they construct a representation of the prefix $y_{<i}$ used to predict $y_i$. NADE accumulates contributions from previous inputs through a shared masked network. RNNs repeatedly transform a fixed-size state as new elements arrive — compressing the entire past into a single vector before knowing what future steps will need. Transformers compute context dynamically: rather than compressing, each step directly retrieves whatever past information is most relevant through attention.

The conceptual shift is from compression to retrieval. An RNN must commit to a summary of the past before seeing the future. Attention postpones that commitment indefinitely — each new position queries the full available history and pulls out exactly what it needs.
