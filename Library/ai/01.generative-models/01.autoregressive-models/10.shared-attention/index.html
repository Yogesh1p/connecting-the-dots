The Inference Bottleneck

The previous article examined the quadratic cost of attention during training: computing $QK^\top$ for a sequence of length $n$ requires $O(n^2)$ operations. Approximation methods — sparse patterns, low-rank decompositions, kernel methods — address this by reducing which interactions are computed.

At inference time the bottleneck is different. Generation is autoregressive: tokens are produced one at a time, left to right. At each step, the model must attend to every token generated so far. Rather than recomputing the key and value vectors for all previous positions from scratch at every step, Transformers cache them. This is the KV cache.

The KV cache stores, for each layer, the key matrix $K$ and value matrix $V$ built from all tokens generated so far. When generating token $t+1$, the model computes only the new query vector $q_{t+1}$, retrieves the cached $K$ and $V$, and runs attention. This converts the per-step cost from $O(t^2)$ to $O(t)$, making generation tractable.

But the KV cache has its own cost: memory. For a model with $L$ layers, $H$ attention heads, head dimension $d$, and a sequence of length $n$, the cache holds

$$2 \cdot L \cdot H \cdot d \cdot n$$

values — the factor of two accounting for both keys and values. For large models, this is substantial. With $L = 32$, $H = 32$, $d = 128$, and $n = 8192$, stored in half-precision (2 bytes), the KV cache occupies roughly 4 GB for a single sequence. Batching multiple sequences multiplies this by the batch size.

This memory pressure limits throughput in two ways. First, it constrains the batch size: more cache memory per sequence means fewer sequences can run simultaneously. Second, at each generation step, the GPU must load the entire cache from memory to compute attention — and memory bandwidth, not arithmetic throughput, is often the binding constraint at inference time.

The question shared attention methods address is direct: does every head in every layer need its own independent $K$ and $V$ projections?

Multi-Head Attention: The Baseline

In standard multi-head attention (MHA), the $H$ heads are fully independent. For each head $h$, separate projection matrices $W_Q^{(h)}$, $W_K^{(h)}$, and $W_V^{(h)}$ transform the input into distinct query, key, and value spaces. Each head computes its own attention independently, and the results are concatenated.

Definition: Multi-Head Attention (MHA)

Let $X \in \mathbb{R}^{n \times d_{\text{model}}}$ be a sequence of token representations and $H$ the number of heads, each of dimension $d = d_{\text{model}} / H$.

For each head $h = 1, \dots, H$:

$$Q^{(h)} = X W_Q^{(h)}, \quad K^{(h)} = X W_K^{(h)}, \quad V^{(h)} = X W_V^{(h)}$$

where $W_Q^{(h)}, W_K^{(h)}, W_V^{(h)} \in \mathbb{R}^{d_{\text{model}} \times d}$. Each head computes

$$\text{head}_h = \mathrm{Attention}(Q^{(h)}, K^{(h)}, V^{(h)}).$$

The outputs are concatenated and projected:

$$\mathrm{MHA}(X) = \mathrm{Concat}(\text{head}_1, \dots, \text{head}_H)\,W^O.$$

The KV cache stores $H$ key matrices and $H$ value matrices per layer.

At each generation step, all $2H$ cached matrices must be loaded to produce the next token. This is the quantity that shared attention methods target.

Multi-Query Attention: One KV Head for All

The most aggressive reduction is to share a single set of key and value projections across all query heads. This is multi-query attention (MQA), introduced by Shazeer in 2019.

In MQA, each of the $H$ heads has its own query projection, but all heads share one key projection $W_K$ and one value projection $W_V$. The key and value vectors computed from the input are the same for every head.

Definition: Multi-Query Attention (MQA)

Let $X \in \mathbb{R}^{n \times d_{\text{model}}}$ with $H$ query heads and a single shared KV head.

Compute shared keys and values once:

$$K = X W_K, \quad V = X W_V, \quad W_K, W_V \in \mathbb{R}^{d_{\text{model}} \times d}.$$

For each query head $h = 1, \dots, H$:

$$Q^{(h)} = X W_Q^{(h)}, \quad \text{head}_h = \mathrm{Attention}(Q^{(h)}, K, V).$$

The outputs are concatenated and projected:

$$\mathrm{MQA}(X) = \mathrm{Concat}(\text{head}_1, \dots, \text{head}_H)\,W^O.$$

The KV cache stores only $1$ key matrix and $1$ value matrix per layer, regardless of $H$.

The KV cache is reduced by a factor of $H$. For a model with 32 heads, the cache is 32 times smaller. The bandwidth cost per generation step is reduced by the same factor.

The question is whether this sharing degrades quality. Empirically, the answer is: somewhat, but less than expected. Different query heads still learn different projections and produce different attention patterns, even when attending over the same keys and values. The keys and values determine what can be attended to; the queries determine what is actually attended to. Diversity among heads is preserved through the query projections alone.

In practice, MQA models trained from scratch perform slightly below comparable MHA models but achieve significantly higher inference throughput. This trade-off is often acceptable in production settings where throughput is the primary constraint.

Grouped-Query Attention: A Middle Ground

Multi-query attention is the most aggressive form of KV sharing, but it discards all KV diversity. A softer alternative is to share keys and values within groups of heads rather than across all heads simultaneously. This is grouped-query attention (GQA).

In GQA, the $H$ query heads are partitioned into $G$ groups of equal size $H/G$. Each group shares one key projection and one value projection. The KV cache stores $G$ key matrices and $G$ value matrices per layer — a factor of $H/G$ reduction compared to MHA.

Definition: Grouped-Query Attention (GQA)

Partition the $H$ query heads into $G$ groups, each of size $H/G$. Assign each group $g$ its own key and value projections $W_K^{(g)}$ and $W_V^{(g)}$.

For group $g$, compute shared keys and values:

$$K^{(g)} = X W_K^{(g)}, \quad V^{(g)} = X W_V^{(g)}.$$

For each query head $h$ in group $g$:

$$Q^{(h)} = X W_Q^{(h)}, \quad \text{head}_h = \mathrm{Attention}(Q^{(h)}, K^{(g)}, V^{(g)}).$$

The outputs are concatenated and projected:

$$\mathrm{GQA}(X) = \mathrm{Concat}(\text{head}_1, \dots, \text{head}_H)\,W^O.$$

The KV cache stores $G$ key matrices and $G$ value matrices per layer.

  • $G = H$: recovers MHA (no sharing).
  • $G = 1$: recovers MQA (all heads share one KV).
  • $1 < G < H$: interpolates between the two extremes.

GQA offers a continuous trade-off between model quality and inference efficiency. Models like Llama 2 70B and Mistral 7B use GQA with $G = 8$ groups, preserving meaningful KV diversity while substantially reducing cache size and bandwidth requirements. The key empirical finding is that intermediate groupings — rather than the extremes — often match MHA quality closely while recovering most of MQA's efficiency gains.

Comparing the Head-Sharing Strategies

The three methods form a clean hierarchy along a single axis: how many distinct KV heads are maintained.

Method KV heads per layer Cache size (relative) Quality Throughput
MHA $H$ $1\times$ (baseline) Highest Lowest
GQA $G$, where $1 < G < H$ $G/H \times$ Near-MHA Near-MQA
MQA $1$ $1/H \times$ Slightly below MHA Highest

In all three cases, the number of query heads $H$ is unchanged. The computation performed by the model — the expressiveness of the queries — is not reduced. Only the key and value representations are shared, reducing what must be stored and loaded from memory at each inference step.

Uptraining: Converting MHA to GQA

A practical concern is that most large models are initially trained with standard MHA. Retraining from scratch with GQA is expensive. An alternative is uptraining: take a pretrained MHA model, convert it to GQA by collapsing its KV heads, and continue training for a small number of steps to let the model adapt.

The conversion is straightforward. If $H$ heads are to be collapsed into $G$ groups, the $H/G$ key (or value) projection matrices within each group are mean-pooled into a single matrix:

$$W_K^{(g)} = \frac{1}{H/G} \sum_{h \in \text{group } g} W_K^{(h)}.$$

This initialisation is better than random because it preserves the average behaviour of the original heads. A short uptraining run — typically a small fraction of the original training budget — then fine-tunes the resulting model. The GQA paper found that models uptraining on 5% of the original training tokens recover nearly all of the quality lost by the conversion, while retaining the full inference efficiency of GQA.

Cross-Layer Attention: Sharing Across Layers

MQA and GQA share KV projections across heads within a single layer. A different axis of sharing is across layers: multiple layers reuse the same KV representations, computed once and cached once.

This is the idea behind cross-layer attention (CLA). In CLA, consecutive layers are grouped into blocks. Within each block, only the first layer computes full key and value projections. Subsequent layers reuse the KV outputs of the first layer, contributing no new KV cache entries of their own.

Definition: Cross-Layer Attention (CLA)

Partition the $L$ layers into blocks of size $b$. Within block $\ell$, the first layer $\ell_0$ computes key and value projections normally:

$$K_{\ell_0} = X_{\ell_0} W_K^{(\ell_0)}, \quad V_{\ell_0} = X_{\ell_0} W_V^{(\ell_0)}.$$

Subsequent layers $\ell_1, \dots, \ell_{b-1}$ in the block reuse these projections:

$$K_{\ell_i} = K_{\ell_0}, \quad V_{\ell_i} = V_{\ell_0} \quad \text{for } i = 1, \dots, b-1.$$

Each layer still computes its own query projections and output projections. Only the KV cache entries are shared.

Total KV cache entries per sequence: $L/b$ (one per block) instead of $L$.

The cache reduction from CLA is multiplicative with the reduction from GQA or MQA. A model using GQA with $G$ groups and CLA with block size $b$ stores

$$\frac{G}{H} \cdot \frac{1}{b}$$

of the KV cache required by standard MHA. These two forms of sharing are orthogonal and can be combined freely.

The key question for CLA is whether reusing KV representations from a lower layer compromises the expressiveness of higher layers. The layers within a block still have distinct query projections and distinct feedforward sub-layers; only the keys and values are shared. Empirically, models with block size $b = 2$ recover nearly all of the quality of standard models while halving the KV cache. Larger block sizes degrade quality more noticeably, suggesting that using stale KV representations across many layers limits the model's ability to represent the context at each level of abstraction.

Why KV Sharing Works

It is worth pausing to understand why sharing key and value projections causes so little quality loss relative to sharing query projections.

Queries and keys together determine the attention pattern — which positions attend to which. Values determine what information is retrieved once the pattern is fixed. Diversity in queries is sufficient to produce diversity in attention patterns even when all heads attend over the same set of keys. Each head can focus on a different subset of positions, capture a different relationship, and retrieve different information — as long as its query is distinct.

By contrast, if queries were shared and keys were distinct, different heads would select similar positions (because the queries drive the selection) but extract different information from them. This is a less useful form of diversity: diversity in what is retrieved from the same positions, rather than diversity in which positions are selected.

The asymmetry explains the empirical findings. Sharing KV projections while maintaining distinct queries preserves the most important source of diversity in multi-head attention: variability in which parts of the context each head attends to.

The Interaction with Rotary Position Embeddings

Most modern Transformer models use rotary position embeddings (RoPE) rather than absolute position embeddings. RoPE encodes position by rotating the query and key vectors before the dot product, such that the attention score between positions $i$ and $j$ depends only on their relative offset $i - j$.

Rotary embeddings are applied to queries and keys, but not to values. This has a direct implication for KV sharing. When keys are shared across heads or layers, the position encoding is applied once and shared. Because RoPE encodes relative positions in the dot product itself — not in a separate positional vector added to the representation — the shared keys carry the correct positional information for each head that attends over them. There is no inconsistency in sharing rotary-encoded keys.

For cross-layer attention, the situation requires more care. A key computed at layer $\ell_0$ uses the token representations at that layer. When reused at layer $\ell_1$, those representations may have drifted from the current state. The positional encoding remains correct — positions do not change across layers — but the semantic content of the key may not reflect the refined representation produced by the intervening feedforward sub-layer. This is the fundamental cost of cross-layer sharing: positional consistency is maintained, but semantic freshness is not.

Shared Attention in Practice

The adoption of KV-sharing methods in production models reflects their favourable quality–efficiency trade-off.

GQA with a modest number of groups — typically 8 — has become the de facto standard for large open-weight models. Llama 2 70B, Llama 3 8B and 70B, Mistral 7B, Mixtral 8×7B, and Gemma all use GQA. The consistent pattern across these models is that quality is nearly indistinguishable from MHA at the scales where it has been tested, while throughput improvements are large and directly measurable.

MQA (with a single KV head) appears in some smaller models where inference speed is the primary constraint, such as early versions of the PaLM family. Its larger quality gap relative to MHA makes it less common in flagship models.

Cross-layer attention is less widely adopted as of writing but has been demonstrated to reduce KV cache memory by a further factor of two with $b = 2$, and models combining CLA with GQA have been shown to match the quality of GQA-only models at reduced memory cost.

The Broader Principle

Shared attention methods do not change what attention computes. They reduce what attention stores and loads. The attention mechanism remains the same weighted aggregation of values according to query-key compatibility. The change is purely in how many distinct key and value representations are maintained — across heads within a layer, or across layers within a model.

This is a different kind of approximation from those discussed in the previous article. Sparse attention, low-rank methods, and kernel approximations reduce the computational cost of the attention operation itself — the number of interactions that are evaluated. Shared attention reduces the memory footprint and bandwidth cost associated with caching and retrieving the representations that attention operates on.

The two families of methods are complementary. A model can use FlashAttention (IO-aware exact computation) to train efficiently, GQA (head sharing) to reduce the KV cache during inference, and sparse attention patterns (for very long contexts) to extend the sequence length that is tractable. Each addresses a different aspect of the attention bottleneck, and they can be combined.

The result is a practical toolkit for deploying attention-based models at scale: exact in computation, efficient in memory, and flexible in how the sharing is configured to match the demands of the task and the constraints of the hardware.
